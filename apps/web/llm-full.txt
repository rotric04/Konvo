## 1. Monorepo Topography & Components

Konvo.Space is structured as a Python-based monorepo containing multiple independent microservices, packaged library utilities, and a front-end single-page application (SPA).

- **apps/web/**: Static assets (HTML, CSS, Vanilla JS) served at the root `/`.
  - `index.html`: Main dashboard covering all views (Discovery, Deck, Chat, Resonance Grid, Twin DNA, Settings, Feedback).
  - `auth.html`: JWT-based user register/login page.
  - `feedback.html`: Feedback collection interface.
  - `style.css`: Premium dark obsidian custom styling sheet.
  - `app.js`: Master front-end control script coordinating UI transitions, WebSockets, Three.js grids, MapLibre tiles, and API communication.
- **services/**: Independently runnable microservices.
  - `auth-service/` (Port 8001): User register, login, OTP verification, and JWT issuance.
  - `user-service/` (Port 8002): Profiles management, digital fingerprints, and birth-based astrology computations.
  - `behavior-engine/` (Port 8003): Personality profile evaluations and match compatibility scores.
  - `sentiment-engine/` (Port 8004): Live message and community sentiment analyzers running over WebSockets.
  - `ai-agent-service/` (Port 8005): Twin AI agents, customizable prompts, and messaging pipelines.
  - `community-service/` (Port 8006): Health indices and activity stats for groups.
  - `discussion-service/` (Port 8007): User-generated posts and comments with sentiment tagging.
  - `graph-service/` (Port 8008): Real-time user affinity network nodes/edges using WebSocket pipelines.
  - `search-service/` (Port 8010): Dynamic index search across users, posts, and groups.
  - `feedback-service/` (Port 8011): Direct feedback collection and storage.
- **packages/**: Shared utilities.
  - `shared-utils/`: Database sessions (`database.py`), ORM entities (`models.py`), and helper functions.
  - `shared-schemas/`: Pydantic validation schemas (`schemas.py`).
- **infrastructure/**: Deployment configurations (Nginx gateway, Docker, etc.).

## 2. API Routes Catalog

All API endpoints are aggregate-mounted onto the primary Gateway API on port `8000`.

### Auth Service APIs
- `POST /api/auth/register`: Register a new account.
- `POST /api/auth/login`: Authenticate and obtain JWT token.
- `POST /api/auth/verify-otp`: Validate 6-digit OTP code.
- `GET /api/auth/me`: Fetch authenticated user credentials.

### User Service APIs
- `GET /api/users/profile`: View specific user profile details.
- `PUT /api/users/profile`: Modify current profile parameters.
- `GET /api/users/astrology`: Return moon/ascendant computations.

### Behavior Engine APIs
- `GET /api/compatibility/swipe`: Retrieve next candidate cards.
- `POST /api/compatibility/swipe`: Submit pass/like swipe decisions.
- `GET /api/compatibility/resonance`: Get compatibility score between two users.

### Sentiment Engine APIs
- `GET /api/sentiment/live`: Return real-time sentiment distribution.
- `WS /api/sentiment/ws/live-sentiment`: WebSocket connection streaming global sentiment metrics.

### AI Agent Service APIs
- `GET /api/agents/twin`: Retrieve active AI twin settings.
- `POST /api/agents/twin`: Create/modify custom AI twin prompt.
- `POST /api/agents/chat`: Chat with another user's AI twin.

### Graph Service APIs
- `GET /api/graph/topology`: Fetch node/edge JSON data of connection grid.
- `WS /api/graph/ws/live-graph`: WebSocket for real-time nodes updates.

### Community & Feedback APIs
- `GET /api/communities/health`: Return community statistics and scores.
- `POST /api/feedback/submit`: Submit app feedback.

## 3. Database Schema Blueprint

The database is built on SQLAlchemy mapping to SQLite/PostgreSQL. Key tables include:

### User (`users`)
- `id` (Integer, Primary Key)
- `email` (String, Unique, Indexed)
- `hashed_password` (String)
- `konvo_id` (String, Unique)
- `role` (String)
- `otp_secret` (String)
- `otp_verified` (Boolean)
- `premium_user` (Boolean)
- `created_at` (DateTime)

### Profile (`profiles`)
- `id` (Integer, Primary Key)
- `user_id` (ForeignKey)
- `display_name` (String)
- `gender` (String)
- `digipin` (String)
- `mbti_type` (String)
- `bio` (Text)
- `birth_date` (DateTime)
- `birth_place` (String)

### BehavioralLedger (`behavioral_ledgers`)
- `id` (Integer, Primary Key)
- `user_id` (ForeignKey)
- `empathy_index` (Float)
- `curiosity_index` (Float)
- `listening_index` (Float)
- `trust_index` (Float)
- `updated_at` (DateTime)

## 4. Security & SEO Standards

- **robots.txt**: Restricts generic scrapers while maintaining visibility for primary AI crawlers (Claude, Gemini, Google, Perplexity).
- **security.txt**: Exposes security reporting pathways under RFC 9116 specification.
- **CSP & Headers**: Content Security Policy enforces strict origin rules (`default-src 'self' *`). Frames are denied.
